1-Options d'environnement de développement

Dans ce tutoriel, nous nous concentrons sur l’installation ou l’activation des quatre outils fondamentaux de développement logiciel suivants ( Figure 1 ) :

  • 1-Terminal de ligne de commande (« shell »)
  • 2-Éditeur de texte
  • 3-Contrôle de version (Git)
  • 4-Langages de programmation (Ruby, etc.)

Pour plus d'informations sur ces différents types d'applications logicielles, consultez Apprendre suffisamment de ligne de commande pour être dangereux , Apprendre suffisamment d'éditeur de texte pour être dangereux , Apprendre suffisamment de Git pour être dangereux et Apprendre suffisamment de Ruby pour être dangereux .

Figure 1 : Éléments typiques d’un environnement de développement.

Lors de la mise en place d'un environnement de développement, nous recommandons deux possibilités principales, classées par ordre croissant de difficulté :

  • IDE Cloud
  • Système d'exploitation natif (macOS, Linux, Windows)

Si vous êtes relativement inexpérimenté, nous vous recommandons de commencer par l'IDE cloud ( Section 2 ), car il présente le processus de configuration le moins difficile.

Système natif

Bien que l'option IDE soit idéale lorsque vous débutez, il est finalement important de pouvoir développer des logiciels sur votre système d'exploitation natif (OS).Malheureusement, la mise en place d’un environnement de développement natif entièrement fonctionnel peut être un processus difficile et frustrant 1 — laissant probablement de nombreuses occasions d’exercer votre sophistication technique ( Encadré 1 ) — mais c’est un rite de passage essentiel pour tout aspirant génie technique.

Afin de relever ce défi difficile, dans la section 3, nous discuterons de la configuration native du système d'exploitation pour macOS, Linux et Windows.

Encadré 1. Sophistication technique
La sophistication technique est la capacité à résoudre de manière autonome des problèmes techniques.Dans le cadre de l'installation d'un environnement de développement, cela signifie savoir rechercher sur Google le message d'erreur si quelque chose ne va pas, essayer de quitter et de redémarrer une application (comme le shell de ligne de commande) pour voir si cela résout les problèmes, etc.
Il peut y avoir tellement de choses qui peuvent mal se passer lors de la configuration d'un environnement de développement qu'il existe rarement une solution générale au problème. Il vous suffit de continuer à appliquer votre sophistication technique jusqu'à ce que tout fonctionne.Et si vous êtes coincé, ne vous inquiétez pas trop : à un moment ou à un autre, cela nous arrive à tous.

2 IDE Cloud

L'option d'environnement de développement la plus simple est un IDE cloud , qui est un environnement de développement intégré dans le cloud auquel vous accédez à l'aide du navigateur Web de votre choix.Bien que facile à activer, le système résultant est une machine de développement de qualité industrielle, pas un jouet.De plus, l'IDE cloud fonctionne automatiquement sur plusieurs plates-formes, car tout ce dont vous avez besoin est un navigateur Web ordinaire pour l'utiliser (ce que tous les principaux systèmes d'exploitation fournissent).

Il existe plusieurs options commerciales pour exécuter un IDE cloud, et dans le cadre du développement du didacticiel Ruby on Rails, nous nous sommes associés à Cloud9 (qui fait partie d'Amazon Web Services).L'environnement résultant est approprié au développement Web Ruby on Rails et inclut bien entendu tous les éléments mentionnés dans la section 1 .En particulier, AWS Cloud9 est équipé d’un terminal de ligne de commande et d’un éditeur de texte (y compris un navigateur de système de fichiers), comme illustré dans la Figure 2 .Étant donné que chaque espace de travail Cloud9 fournit un système Linux entièrement fonctionnel, il inclut également automatiquement le système de contrôle de version Git, ainsi que Ruby et plusieurs autres langages de programmation.

3.1 macOS

Le système d'exploitation natif Macintosh, appelé à l'origine Mac OS X et désormais connu simplement sous le nom de macOS , dispose d'une interface utilisateur graphique (GUI) soignée tout en étant construit sur une base Unix solide.Par conséquent, macOS est parfaitement adapté à une utilisation comme environnement de développement pour un programmeur.

Les étapes de cette section constituent plus qu'un simple système minimal ; vous pouvez en fait vous en sortir en faisant beaucoup moins, mais vos trois auteurs utilisent tous macOS eux-mêmes, et nous pensons qu'il est important de ne pas vous léser avec une configuration à moitié cuite.

Terminal et éditeur

Bien que macOS soit livré avec un programme de terminal natif, nous vous recommandons d'installer iTerm , qui inclut diverses améliorations qui le rendent meilleur que la valeur par défaut pour les développeurs et autres utilisateurs techniques.

Nous vous recommandons également d'installer un éditeur de texte de programmeur.Il existe de nombreux excellents choix, mais Sublime Text (abordé dans Learn Enough Text Editor to Be Dangerous ) est un bon point de départ si vous n'avez pas déjà de favori.

Au fait, pour les tutoriels Learn Enough et le tutoriel Rails, il est généralement recommandé d'utiliser le shell Bourne-again (Bash) plutôt que le shell Z par défaut (même si cela n'a souvent pas d'importance).Pour passer votre shell à Bash, exécutez chsh -s /bin/bashla ligne de commande, entrez votre mot de passe et redémarrez votre programme de terminal.Tout message d’alerte résultant peut être ignoré en toute sécurité.Consultez l’article du blog Learn Enough « Utilisation de Z Shell sur Mac avec les didacticiels Learn Enough » pour plus d’informations.

Outils de ligne de commande Xcode

Bien que basé sur Unix, macOS 6 n'est pas livré avec tous les logiciels nécessaires à un environnement de développement approprié.Afin de combler cette lacune, les utilisateurs de macOS doivent installer Xcode , une vaste suite d’outils de développement et de bibliothèques de code créée par Apple.

Xcode nécessitait auparavant un téléchargement de plus de 4 Go de fichiers sources d'installation, mais heureusement, Apple a récemment rendu Xcode incroyablement rapide et facile à installer avec une simple commande en ligne de commande, comme indiqué dans la liste

$ xcode-select --install

Brassage maison

L'étape suivante est techniquement facultative, mais à notre avis, elle est nécessaire pour un environnement de développement macOS véritablement professionnel : à savoir, l'installation de l'excellent gestionnaire de paquets Homebrew .

Vous pouvez considérer un gestionnaire de paquets comme un App Store qui s'exécute sur la ligne de commande et qui est rempli de logiciels open source gratuits.De nos jours, la plupart des distributions Linux sont livrées avec un gestionnaire de paquets natif ( Section 3.2 ), mais par défaut, macOS n'en a pas.Homebrew est l'un des nombreux gestionnaires disponibles dans la communauté open source, mais au fil du temps, il est devenu l'option la plus populaire parmi les développeurs macOS sérieux.

Comme nous le verrons dans la section 3.1.5 , nous utiliserons un programme appelé rbenv pour installer Ruby, qui à son tour sera installé via Homebrew dans la section 3.1.4 .Mais Homebrew lui-même nécessite Ruby, il semble donc que nous soyons tombés sur une dépendance circulaire .Heureusement, macOS est livré avec un système Ruby que nous pouvons utiliser pour démarrer l'installation.Nous utiliserons ce Ruby par défaut pour installer Homebrew, puis nous installerons rbenv et des versions Ruby supplémentaires comme indiqué ci-dessus.

Le programme d'installation de Homebrew est un script Bash et peut être consulté à l'aide du curlprogramme décrit dans Apprendre suffisamment de ligne de commande pour être dangereux .Nous pouvons exécuter le script d'installation de Homebrew en utilisant l' bashexécutable système, qui se trouve dans le /binrépertoire.La commande complète apparaît comme dans la liste 5 .

Liste 5 : Installation du gestionnaire de paquets Homebrew.
$ /bin/bash -c " $( curl -fsSL https://www.learnenough.com/homebrew.sh ) "

Notez que la liste 5 utilise une URL de transfert learnenough.com, qui pointe vers le script d'installation Homebrew actuel.De cette façon, si Homebrew modifie l'URL utilisée pour héberger le script, nous pouvons simplement mettre à jour l'adresse de transfert et ce tutoriel continuera à fonctionner comme écrit.(Vous pouvez également simplement copier et coller la commande complète depuis la page d'accueil de Homebrew si vous le souhaitez.)

Homebrew installe une brewcommande de ligne de commande utilisée pour installer, mettre à jour et supprimer des packages.Une fois l'installation de Homebrew terminée, c'est une bonne idée d'exécuter brew doctor, ce qui garantit que tous les répertoires et autorisations nécessaires à Homebrew pour gérer les fichiers locaux sont correctement configurés :

docteur
brasseur

Si vous rencontrez des problèmes à ce stade, vous devrez vous référer au wiki de dépannage de Homebrew , mais vous ne devriez vraiment pas le faire à moins que vous n'ayez apporté des modifications à des dossiers système et des autorisations aléatoires.

Environnement Ruby (rbenv)

Comme nous venons de le voir, macOS est livré avec Ruby préinstallé, mais nous n'avons aucun contrôle sur la version exacte, et macOS ne nous permet pas nativement d'utiliser plusieurs versions de Ruby en parallèle.Pour nous donner plus de flexibilité avec notre environnement de développement, nous allons installer rbenv , qui est un utilitaire qui gère différentes versions de Ruby et s'assure que les packages logiciels Ruby (appelés gems ) sont placés au bon endroit pour que Ruby les trouve.


L'utilisation de rbenv avec la commande ruby-build associée nous permet également de spécifier une version différente de Ruby pour différents référentiels de projets, ce qui est une tâche courante dans le développement de logiciels .Par exemple, une ancienne version d’un programme peut nécessiter une ancienne version de Ruby pour fonctionner correctement.L'utilisation de rbenv signifie que nous pouvons prendre en charge un programme aussi ancien tout en exécutant une version plus à jour de Ruby pour nos autres projets.

L'installation de rbenv est facile avec Homebrew.Nous pouvons installer à la fois rbenv et ruby-build en une seule étape en utilisant brew install rbenv, comme indiqué dans la liste 6

Liste 6 : Installation de rbenv et ruby-build.
$ brew install rbenv # installe également automatiquement ruby-build

Une fois l'installation de la liste 6 terminée, nous devons faire fonctionner rbenv en utilisant rbenv init, comme indiqué dans la liste 7 .

Remarque : si vous utilisez Zsh à la place, remplacez- .zshrcle par .bash_profiledans cette section.Consultez « Utilisation de Z Shell sur Mac avec les didacticiels Learn Enough » pour plus d’informations.

Liste 7 : Initialisation de rbenv.
$ rbenv init
# Chargez rbenv automatiquement en ajoutant
# ce qui suit à ~/.bash_profile ( ou ~/.zshrc si vous utilisez Zsh ) :
eval "$(rbenv init -)"

Si la liste 7 vous donne un message d’erreur tel que « Aucun fichier ou répertoire de ce type. », essayez de quitter votre programme shell avec Ctrl-D et de le redémarrer, puis réessayez la commande.(Ce type de technique de redémarrage et de nouvelle tentative est une sophistication technique classique ( Encadré 1 ).)

Comme on le voit dans la liste 7 , l'exécution rbenv initnous donne une suggestion sur la façon d'éviter d'avoir à initialiser rbenv à la main à chaque fois : nous devons simplement ajouter la ligne

eval "$(rbenv init -)"

à notre fichier de profil Bash .bash_profile(qui est abordé dans Apprendre à utiliser suffisamment l'éditeur de texte pour être dangereux ).

Si vous préférez, vous pouvez utiliser un éditeur de texte pour ajouter la evalligne au fichier de votre système .bash_profile, mais le moyen le plus simple est d'utiliser echol'opérateur d'ajout >> décrit dans Apprendre une ligne de commande suffisamment dangereuse , comme ceci :

$ echo 'eval "$(rbenv init -)"' >> ~/.bash_profile # ou ~/.zshrc si vous utilisez Zsh

Notez que nous avons inclus le répertoire personnel ~dans le chemin afin qu'il fonctionne quel que soit le répertoire dans lequel nous nous trouvons actuellement.

Enfin, pour activer le nouveau fichier de profil, nous devons le faire source(comme mentionné dans Learn Enough Text Editor to Be Dangerous ) :

$ source ~/.bash_profile # ou ~/.zshrc si vous utilisez Zsh

Nouvelle version de Ruby

Maintenant que rbenv est configuré, donnons-lui une version non système de Ruby à gérer.Le processus d'installation est entièrement géré par rbenv, il vous suffit donc de lui indiquer la version que vous souhaitez sur votre système en lui transmettant le numéro de version exact de Ruby.

Nous utiliserons Ruby 3.1.2 dans ce tutoriel, qui, au moment de la rédaction de cet article, fonctionne avec une grande variété d'applications Ruby, mais vous pouvez également utiliser une version actuelle ou précédente comme indiqué sur le site Web de Ruby.

Pour installer la version souhaitée de Ruby en utilisant rbenv, exécutez simplement la commande indiquée dans la liste 8 .Si votre système se plaint que la version Ruby donnée n'est pas disponible, vous devrez mettre à jour votre système pour accéder à la dernière version ( Encadré 2 ).

Liste 8 : Installation d’une nouvelle copie de Ruby.
$ rbenv install 3 .1.2

Après avoir exécuté la commande de la liste 8 , vous devriez voir rbenv démarrer le processus de téléchargement et installer toutes les dépendances nécessaires pour cette version spécifique de Ruby (ce qui peut prendre un certain temps en fonction de la bande passante et des limitations du processeur).

Remarque : vous pouvez recevoir un message d’erreur du formulaire

Ruby-build : définition introuvable : 3.1.2.
Consultez toutes les versions disponibles avec « rbenv install --list ».
Si la version souhaitée est manquante, essayez de mettre à jour Ruby-build :
brew update && brew upgrade Ruby-build.

Si cela se produit, vous devez cloner la ruby-buildliste la plus récente dans le pluginsrépertoire rbenv :

$ mkdir ~/.rbenv/plugins
$ git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build

Si ruby-buildc'est déjà le cas, vous pouvez simplement pullentrer les dernières modifications :

$ cd ~/.rbenv/plugins && git pull && cd -

Encadré 2. Mise à jour et mise à niveau
Bien que nous venions d'installer de nouvelles versions de tous les logiciels pertinents, après un certain temps, votre système local ne sera plus synchronisé avec les dernières versions de Homebrew et tous les packages installés.Afin de mettre à jour le système, il est judicieux de mettre à jour Homebrew lui-même de temps en temps, puis de mettre à niveau les packages installés comme suit :

$ brew mise à jour $ brew mise à niveau

Une fois l'installation de Ruby terminée, nous devons indiquer au système qu'il existe une nouvelle version de Ruby en utilisant la rehashcommande au nom obscur :

$ rbenv rehash

Pour ce guide, nous allons également définir la version Ruby de la liste 8 comme valeur par défaut globale afin que vous n'ayez pas à vous soucier de spécifier la version Ruby lorsque vous démarrez votre projet.La façon de faire cela est d'utiliser la globalcommande :

$ rbenv global 3 .1.2

À ce stade, c'est probablement une bonne idée de redémarrer votre programme shell pour vous assurer que tous les paramètres sont correctement mis à jour.

Pour les travaux futurs, vous souhaiterez peut-être utiliser des versions spécifiques de Ruby pour chaque projet, ce qui peut être fait en créant un fichier appelé .ruby-versiondans le répertoire racine du projet et en incluant la version de Ruby à utiliser.(S'il n'est pas déjà présent sur votre système, vous devrez également l'installer en utilisant rbenv install comme dans la liste 8. )Consultez la documentation de rbenv pour plus d'informations.

Enfin, lors de l'installation d'un logiciel Ruby via des gemmes ou des packages autonomes de code Ruby, il est souvent pratique d'ignorer l'installation de la documentation locale, qui peut prendre plus de temps à installer que le logiciel lui-même, et dans tous les cas, il est plus pratique d'y accéder en ligne.L'empêchement de l'installation de la documentation peut être effectué au cas par cas, mais il est plus pratique d'en faire une valeur par défaut globale en créant un fichier dans le répertoire personnel appelé .gemrcavec des instructions pour ignorer les fichiers de documentation Ruby rarement utilisés (et longs à installer), comme indiqué dans la liste 9 .

Liste 9 : Configuration du .gemrcfichier pour ignorer l’installation de la documentation Ruby.
$ echo "gem: --no-document" >> ~/.gemrc

Avec cette configuration, toutes les utilisations gem install pour installer des gemmes Ruby seront automatiquement simplifiées, rationalisées et sans documentation.

Python

Pour installer Python à l’aide de Homebrew, utilisez la commande suivante :

$ brew install python@3.10

Cela ajoute python3des pip3exécutables (ce dernier est un gestionnaire de paquets Python) :

$ quel python3
/opt/homebrew/bin/python3
$ quel pip3
/opt/homebrew/bin/pip3

De nombreuses instructions en ligne nécessitent que vous exécutiez pythonou pip; dans de tels cas, vous devez ajouter la fin 3afin d'obtenir le résultat attendu.

Git

Une version récente du système de contrôle de version Git devrait être fournie automatiquement avec les outils de ligne de commande Xcode installés dans la section 3.1.2 , ce que vous pouvez vérifier à l'aide de la whichcommande :

$ quel git

Si le résultat est vide, cela signifie que Git n'est pas installé et que vous pouvez l'installer avec Homebrew :

$ brew installer git

Dans tous les cas, vous devez vérifier que le numéro de version de Git est au moins2.28.0 :

$ git --version
git version 2.31.1 # doit être au moins 2.28.0

Si le numéro de version n'est pas assez récent, vous devez exécuter brew updatepuis exécuter brew upgrade git.

3.2 Linux

En raison des origines hautement techniques de Linux, les systèmes Linux sont généralement bien équipés d'outils de développement.Par conséquent, la configuration d’un système d’exploitation Linux natif comme environnement de développement est particulièrement simple.

Chaque distribution Linux majeure est livrée avec un programme de terminal, un éditeur de texte et Git.Il n'y a que quatre étapes principales que nous recommandons en plus des valeurs par défaut :
1-Téléchargez et installez Sublime Text si vous n'avez pas déjà d'éditeur favori.Sublime Text est un produit commercial mais peut être utilisé indéfiniment gratuitement en mode évaluation.
2-Suivez les instructions d'installation de rbenv sur le site Web de rbenv.
3-Installez et configurez Ruby comme indiqué dans la liste 8 et la liste 9 .
4-Installez Python comme indiqué dans la liste 2 .En fonction de votre système exact, vous devrez peut-être utiliser un gestionnaire de paquets autre que apt; dans ce cas, utilisez votre sophistication technique ( encadré 1 ) pour déterminer ce qu'il faut faire.
